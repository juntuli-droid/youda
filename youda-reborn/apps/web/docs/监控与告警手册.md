# 监控与告警手册

## 1. 监控选型

- 错误监控：Sentry
- 结构化日志：应用标准输出 JSON 日志
- 日志采样：`info` 1%，`warn/error` 100%
- 日志保留：建议接入云日志平台保留 30 天

## 2. 已接入能力

- 全局前端异常：`src/app/global-error.tsx`
- Node/Edge 运行时异常上报：`sentry.server.config.ts`、`src/instrumentation.ts`
- 认证路由异常捕获：`src/lib/monitoring.ts`
- 登录、注册、资料接口结构化日志：`src/lib/logger.ts`

## 3. Sentry 仪表盘

- 生产地址：`https://sentry.io/organizations/<your-org>/projects/<your-project>/`
- 认证链路筛选建议：`route.event:auth.*`
- 错误趋势看板：按 `environment`, `route.path`, `user.id` 维度拆分

## 4. 告警规则

| 指标 | 阈值 | 通知渠道 |
|---|---|---|
| P99 响应时间 | `> 2s` 持续 5 分钟 | Slack + 邮件 |
| 5xx 错误率 | `> 1%` 持续 5 分钟 | Slack + 邮件 |
| 内存使用率 | `> 80%` 持续 10 分钟 | Slack + 邮件 |

## 5. Slack 与邮件配置

- `SLACK_WEBHOOK_URL`：Slack Incoming Webhook
- `ALERT_EMAIL_TO`：接收告警汇总的邮箱
- 推荐在 Sentry 中配置 Metric Alert 与 Issue Alert，通知到 Slack 和邮件组

## 6. 每周错误趋势报告模板

### 周报字段

- 周期
- 总错误数
- 影响用户数
- 新增高频错误 Top 5
- 已修复问题数
- 未关闭高优先级问题数
- P95 / P99 响应时间
- 下周重点治理项

### 示例结构

```text
时间范围：2026-04-06 ~ 2026-04-12
总错误数：128
影响用户数：42
新增高频错误 Top 5：auth.login.failed, auth.refresh.failed, profile.update.failed ...
已修复问题数：7
未关闭高优先级问题数：2
P95 / P99：420ms / 1.82s
下周重点治理项：登录刷新失败回溯、Redis 抖动告警去噪
```
