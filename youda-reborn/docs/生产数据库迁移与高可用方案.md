# 生产数据库迁移与高可用方案

## 1. 目标

- 生产库结构与 Prisma migration 完全一致
- 数据迁移过程零丢失、可回滚、可审计
- 支持主从复制、读写分离、自动备份与灾难恢复
- 具备慢查询治理、连接池治理和容量监控

## 2. 迁移流程

1. 在 staging 使用 `DIRECT_DATABASE_URL` 执行 `pnpm --filter @youda/database migrate:deploy`
2. 对比 `packages/database/prisma/migrations` 与目标库 `information_schema`
3. 使用只读快照导出旧数据，执行结构校验和抽样数据校验
4. 生产切换窗口前冻结高风险写入任务
5. 执行生产迁移，随后执行校验脚本与业务冒烟
6. 启用监控观察 30 分钟，无异常后解除冻结

## 3. 主从复制与读写分离

- 主库：写请求与强一致查询
- 从库：个人主页读取、历史列表、统计查询
- 应用层建议：
  - `DATABASE_URL` 指向写库代理或统一连接池入口
  - 读流量后续拆分到 `DATABASE_READ_URL`
- 切换策略：
  - 主库故障时通过代理提升从库为新主库
  - Prisma 迁移始终走 `DIRECT_DATABASE_URL`

## 4. 备份与灾难恢复

- 全量备份：每日 1 次，保留 7-30 天
- Binlog 增量备份：5 分钟内可恢复
- 恢复演练：
  - 每月至少 1 次恢复到临时实例
  - 验证用户、会话、消息和职业数据完整性
- RTO 目标：30 分钟
- RPO 目标：5 分钟

## 5. 性能治理

- 慢查询阈值：`> 200ms`
- 必看指标：
  - 连接数
  - QPS / TPS
  - 锁等待
  - 慢查询 Top N
  - P95 / P99 查询耗时
- 已落地索引见：
  - [数据库ER与索引设计.md](/Users/lijuntu/Desktop/youda/youda-reborn/packages/database/docs/数据库ER与索引设计.md)

## 6. 连接池治理

- 运行态连接走连接池代理
- Prisma Client 单例缓存，避免热实例重复建连
- Serverless 冷启动基准脚本：
  - `pnpm --filter @youda/database benchmark:serverless`

## 7. 数据完整性校验

- 行数校验：`users/friends/matches/chat_messages/vlogs/careers`
- 唯一性校验：`users.email/users.username/refresh_tokens.tokenHash`
- 外键校验：好友关系、消息归属、会话与挑战令牌
- 抽样冒烟：注册、登录、刷新、更新资料、创建 Vlog、添加生涯
