# 密钥治理与审计方案

## 1. 目标

- 所有敏感变量均通过密钥管理服务或 CI Secret 注入
- 不在仓库、镜像、构建日志中暴露明文
- 所有敏感变量访问具备审计记录

## 2. 推荐架构

### 方案 A：HashiCorp Vault

- 使用 Vault KV 存储：
  - `DATABASE_URL`
  - `DIRECT_DATABASE_URL`
  - `JWT_SECRET`
  - `JWT_REFRESH_SECRET`
  - `REDIS_URL`
  - `SENDGRID_API_KEY`
- 应用通过短期 Token 拉取配置
- 启用 Audit Device 记录访问轨迹

### 方案 B：AWS Secrets Manager

- 每个环境一组 Secret
- 通过 IAM Role 授权服务读取
- 开启 CloudTrail 审计访问日志
