# 匹配队列设计文档

## 目标
- 将原本纯前端进度条 mock，替换为 Redis + MySQL 持久化的真实排队服务。
- 支持按游戏、地区、年龄区间、性别偏好、兴趣标签进行兼容度匹配。

## 架构
- Redis：使用 `ZSET` 保存实时排队优先级，用 Lua 保证入队原子性。
- MySQL：`match_queue_entries` 持久化排队状态，`match_logs` 保存匹配审计。
- Next Route Handlers：提供 `/api/match/queue` 与 `/api/match/room/messages` 边界。

## 匹配流程
1. 客户端提交筛选条件，服务端写入 `match_queue_entries`。
2. Redis 写入短 TTL 实时索引，数据库保存完整条件用于补偿。
3. 轮询状态时触发一次轻量匹配扫描。
4. 当兼容度分数 `>= 0.85` 时创建 `matches` 房间并双向更新队列条目。

## 兼容度算法
- 兴趣重叠：35%
- 地区：30%
- 性别偏好：15%
- 年龄区间交集：20%

## 灰度发布建议
- 阶段 1：仅对内部账号开启真实队列，保留前端兜底。
- 阶段 2：5% 流量切入真实队列，重点观察取消率与等待时长。
- 阶段 3：全量开启，并接入 Redis Cluster 与后台定时扫尾任务。

## 监控规则
- 匹配成功率 < 85%
- 平均等待时长 > 30s
- `MATCH_CANCELLED / MATCH_ENQUEUED` 比例异常升高
- Redis `eval` 错误率 > 1%
