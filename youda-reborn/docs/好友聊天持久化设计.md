# 好友/聊天持久化设计

## 范围
- 服务端持久化好友关系、私聊消息、房间消息、消息回执、撤回状态、拉黑与举报审计。
- 前端从本地 `useState` mock 切换为 `/api/social/*` 与 `/api/match/room/messages` 真实接口。

## 核心表
- `friends`：好友申请与双向好友关系。
- `chat_messages`：文本、图片、语音统一消息表，支持 `DIRECT` / `MATCH`。
- `chat_message_receipts`：已送达、已读回执。
- `block_relations`：双向拉黑关系判断。
- `audit_events`：好友、聊天、空态曝光点击的审计与埋点。

## 关键约束
- 时间精度统一 `DATETIME(3)`，保证毫秒级排序。
- 私聊顺序按 `createdAt + id` 排序。
- 客户端可传 `clientMessageId` 去重。
- 消息撤回仅允许发送方操作，保留 `recalledAt` 与 `recallReason`。

## 离线补偿
- 私聊与房间消息接口都支持 `since` 参数。
- 客户端轮询增量同步，断线重连后按最后一条时间戳回补。
- 已读回执单独接口更新，避免和消息内容写入事务互相阻塞。

## 索引
- `chat_messages(senderId, receiverId, createdAt)`
- `chat_messages(receiverId, recalledAt, createdAt)`
- `chat_message_receipts(userId, readAt, createdAt)`
- `friends(userId, status, createdAt)` 与 `friends(friendId, status, createdAt)`

## 当前实现边界
- 已支持两人私聊和两人匹配房间消息。
- 真实图片/语音上传仍需要对象存储接入，当前接口先接收附件 URL。
