# 密钥轮换手册

## 1. 适用范围

本手册适用于以下敏感配置的轮换：

- JWT_SECRET
- JWT_REFRESH_SECRET
- DATABASE_URL
- DIRECT_DATABASE_URL
- REDIS_URL
- SENDGRID_API_KEY
- SENTRY_DSN

## 2. 通用原则

- 所有密钥轮换必须先在 staging 环境验证
- 生产环境轮换必须具备回滚方案
- 轮换前必须确认配置注入位置已更新
- 轮换后必须验证认证、数据库、缓存、邮件与监控链路

## 3. JWT 平滑轮换步骤

### 3.1 轮换前准备

- 生成新的 `JWT_SECRET_NEXT` 与 `JWT_REFRESH_SECRET_NEXT`
- 在配置中心或平台环境变量中预先写入新值
- 保留旧值作为过渡期验证密钥
- 过渡期长度至少覆盖当前 Refresh Token 最长生命周期（默认 30 天）

### 3.2 过渡期策略

- 签发新令牌时使用新密钥
- 校验旧令牌时允许旧密钥和新密钥同时生效
- 过渡期建议保留 7-30 天，覆盖 refresh token 存活周期

### 3.3 切换步骤

- 先更新 staging
- 验证注册、登录、刷新、退出流程
- 再更新 production
- 观察 30 分钟以上的登录失败率与 401 波动
- 过渡期结束前，确认旧 Refresh Token 已自然淘汰或被主动吊销

### 3.4 完成标准

- 新旧会话均可在过渡期正常完成刷新
- 无异常大面积 401/403
- 过渡期结束后移除旧密钥

## 4. 数据库凭据轮换步骤

### 4.1 准备

- 在云数据库创建新账号
- 复制旧账号的最小权限集合
- 同步记录新的 `DATABASE_URL` 与 `DIRECT_DATABASE_URL`

### 4.2 执行

- 在 staging 环境替换 `DATABASE_URL` 与 `DIRECT_DATABASE_URL`
- 运行 Prisma 连接测试、迁移测试、读写验证
- 在 production 环境发布新连接串
- 观察连接池、错误率、查询耗时

### 4.3 回滚

- 保留旧数据库账号一段观察期
- 出现异常立即切回旧 `DATABASE_URL` 与 `DIRECT_DATABASE_URL`
- 排查连接数、权限、白名单等问题后再二次切换

## 5. Redis 凭据轮换步骤

- 在 Redis 服务端生成新密码或新实例
- 更新 `REDIS_URL`
- 验证限流、会话刷新、验证码或缓存链路
- 观察连接超时与命中率

## 6. SendGrid 密钥轮换步骤

- 在 SendGrid 后台生成新 API Key
- 更新 `SENDGRID_API_KEY`
- 发送测试邮件验证模板和发件域
- 确认旧密钥停用前没有投递中断

## 7. Sentry DSN 轮换步骤

- 创建新的项目密钥或 DSN
- 更新 `SENTRY_DSN`
- 触发一条测试错误，验证告警和仪表盘正常

## 8. 发布后检查清单

- 登录成功率正常
- 刷新令牌流程正常
- JWT 旧密钥是否已按计划下线
- 注册和忘记密码流程正常
- 数据库读写无异常
- Redis 限流无超时
- 邮件发送成功
- Sentry 可收到新事件
